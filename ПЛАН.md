# ПЛАН разработки — Витрина Елены

---

## Структура плана

```
Этап 0 — Подготовка
Этап 1 — MVP: Каталог + Заказ + Профиль
Этап 2 — Логистика: Посылки + Треки
Этап 3 — Возвраты
Этап 4 — Шоурум + Вишлист + Уведомления
Этап 5 — Рекомендации + Голос Елены (ИИ)
Этап 6 — Аналитика
```

---

## Этап 0 — Подготовка

**Цель:** выровнять понимание, выбрать технологию, не начать строить лишнее.

### Задачи

- [ ] Провести установочную встречу с Еленой
- [ ] Показать Елене прототип / макет интерфейса
- [ ] Получить её подтверждение, что она готова работать в системе
- [ ] Выбрать технологию: no-code (Вариант A) или разработка (Вариант B)
- [ ] Определить разработчика / исполнителя
- [ ] Зафиксировать все активные заказы Елены в таблице (временная CRM)
- [ ] Собрать список текущих клиентов с базовыми параметрами
- [ ] Собрать список поставщиков Елены
- [ ] Определить формат, в котором каждый поставщик присылает данные

### Результат этапа

- Выбрана технология
- Есть исполнитель
- Елена согласна
- Есть временная таблица с заказами и клиентами

---

## Этап 1 — MVP

**Цель:** зафиксировать заказ, убрать хаос в переписке.

**Ориентировочные сроки:** 4–8 недель после старта

### Модуль 1.1 — Каталог товаров

- [ ] Форма ручного добавления товара (для Елены)
- [ ] Импорт товаров из CSV
- [ ] Карточка товара: фото, видео, размеры, цвета, длина, рост, состав
- [ ] Разделы: Хиты / Новинки / Категории
- [ ] Фильтры: размер, цвет, страна, длина, повод, наличие, цена
- [ ] Статус наличия: в наличии / ожидается / нет

### Модуль 1.2 — Профиль клиента

- [ ] Форма анкеты для клиента (публичная ссылка)
- [ ] Карточка клиента для Елены: параметры, предпочтения, события, заметки
- [ ] Список клиентов с поиском и фильтрами
- [ ] Поле "личные фразы из переписки"
- [ ] Поле "не предлагать" (цвета, фасоны)

### Модуль 1.3 — Форма запроса / подбора

- [ ] Публичная форма: повод, что ищу, размер, рост, цвет, бюджет
- [ ] Елена получает уведомление о новом запросе
- [ ] Запрос привязывается к профилю клиента

### Модуль 1.4 — Карточка заказа

- [ ] Создание заказа вручную Еленой
- [ ] Добавление позиций из каталога
- [ ] Поля позиции: товар, размер, цвет, количество, цена, скидка
- [ ] Статусы позиции: подтверждена / под вопросом / отменена / замена
- [ ] Итоговая сумма (считается автоматически)
- [ ] Поле оплаты: сумма, дата, получатель
- [ ] Статусы заказа (полный список — см. ТЗ)
- [ ] Заметки по заказу

### Модуль 1.5 — Ссылка на заказ для клиента

- [ ] Уникальная ссылка на заказ (без логина)
- [ ] Клиент видит: состав, цены, статус, оплату
- [ ] Клиент НЕ может редактировать — только читать
- [ ] Адаптирована под мобильный телефон

### Критерий готовности этапа 1

> Елена может создать заказ, добавить позиции, отправить клиенту ссылку.
> Клиент открывает ссылку и видит свой заказ.

## Этап 2 — Логистика

**Цель:** прозрачность по посылкам — клиент знает, что и когда едет.

**Ориентировочные сроки:** 3–4 недели после этапа 1

### Задачи

- [ ] Создание посылок внутри заказа
- [ ] Привязка позиций к конкретной посылке
- [ ] Поле трек-номера и перевозчика
- [ ] Статусы посылки: комплектуется / отправлена / доставлена
- [ ] Клиент видит в своей ссылке: что в посылке 1, что в посылке 2
- [ ] Шаблон сообщения об отправке (Елена копирует и отправляет в Telegram)

### Критерий готовности этапа 2

> Клиент, не написав Елене, понимает: что отправлено, что едет отдельно, какой трек.

---

## Этап 3 — Возвраты

**Цель:** возврат — это процесс, а не переговоры.

**Ориентировочные сроки:** 2–3 недели после этапа 2

### Задачи

- [ ] Создание возврата в карточке заказа
- [ ] Выбор позиций для возврата
- [ ] Причина возврата: размер / цвет / качество / передумала
- [ ] Автоматический расчёт суммы к возврату
- [ ] Статусы возврата: запрошен / клиент отправил / получен продавцом / деньги возвращены
- [ ] История частичных возвратов
- [ ] Итоговая сумма заказа пересчитывается после возврата

### Критерий готовности этапа 3

> Елена нажимает "оформить возврат" → система считает сумму → возврат закрывается со статусом.

---

## Этап 4 — Шоурум + Вишлист + Уведомления

**Цель:** превратить визит в шоурум в долгосрочный актив. Запустить уведомления.

**Ориентировочные сроки:** 3–5 недель после этапа 3

### Модуль 4.1 — Вишлист

- [ ] Кнопка "Добавить в вишлист" на карточке товара
- [ ] Кнопка "Уведомить, когда появится" (для товаров не в наличии)
- [ ] Вишлист клиента: список, статус каждой позиции
- [ ] Елена видит: у кого что в вишлисте (по товару / по клиенту)

### Модуль 4.2 — Сессия примерки в шоуруме

- [ ] Создание сессии примерки: клиент + дата + место
- [ ] Добавление товаров к сессии с реакцией (нравится / купила / не подошло)
- [ ] Поле для примеренного размера и комментария по посадке
- [ ] Непонравившееся — не предлагать клиенту снова
- [ ] После сессии — вишлист клиента обновляется автоматически
- [ ] Елена видит сводку после визита: купила / хочет / не подошло

### Модуль 4.3 — Уведомления

- [ ] Настройки уведомлений в профиле клиента (галочки)
- [ ] Выбор канала: email / WhatsApp / Telegram-бот
- [ ] Уведомление: акция на товар из вишлиста
- [ ] Уведомление: товар из вишлиста появился в наличии
- [ ] Уведомление: статус заказа изменился
- [ ] Уведомление: посылка отправлена (трек + состав)

### Критерий готовности этапа 4

> После визита в шоурум клиент получает email: "У вас 5 отложенных вещей. Посмотрите когда будете готовы."
> При акции на товар из вишлиста — клиент получает персональное письмо.

---

## Этап 5 — Рекомендации + Голос Елены

**Цель:** умный персональный маркетинг с голосом Елены, а не безликая рассылка.

**Ориентировочные сроки:** 4–6 недель после этапа 4

### Модуль 5.1 — Похожие товары

- [ ] Связи между товарами (Елена отмечает вручную при добавлении)
- [ ] Типы связей: похожий фасон / похожий цвет / похожий повод / визуальный близнец
- [ ] Система показывает: у кого в вишлисте похожий товар

### Модуль 5.2 — Инструмент "Продвинуть товар"

- [ ] При добавлении нового товара: "На что похож?"
- [ ] Система показывает: сколько клиентов с похожим в вишлисте
- [ ] Кнопка "Подготовить сообщения для этих клиентов"

### Модуль 5.3 — ИИ-черновики сообщений

- [ ] Интеграция с Claude API
- [ ] ИИ берёт из профиля клиента: имя, волосы, предпочтения, события, личные фразы
- [ ] ИИ составляет черновик персонального сообщения
- [ ] Елена видит черновик, правит, отправляет одним нажатием
- [ ] Банк фраз голоса Елены (она добавляет сама, ИИ их использует)
- [ ] Три типа сообщений: транзакционные / персональные / маркетинговые

### Критерий готовности этапа 5

> Елена добавила новый товар → нажала "продвинуть" → увидела черновики для 5 клиентов → отправила с правками → 1–2 клиента написали.

---

## Этап 6 — Аналитика

**Цель:** Елена видит, что работает, что нет.

**Ориентировочные сроки:** после стабилизации системы

### Задачи

- [ ] Топ товаров по количеству заказов
- [ ] Топ товаров в вишлисте (спрос без покупки)
- [ ] Частота возвратов по товарам (какие чаще возвращают)
- [ ] Частота возвратов по причинам (размер / цвет / качество)
- [ ] Ценность клиента (сумма всех заказов)
- [ ] "Спящие" клиенты (не покупали N месяцев)
- [ ] Конверсия: запрос → заказ
- [ ] Средний чек по периодам

---

## Приоритеты при конфликте

Если ресурс ограничен — вот порядок:

1. Карточка заказа + статусы (снимает главную боль)
2. Профиль клиента (один раз — и навсегда)
3. Частичная отправка + трек (прозрачность)
4. Возврат с автоматическим расчётом
5. Вишлист + уведомления об акциях
6. Шоурум / примерочная сессия
7. Рекомендации + ИИ-черновики
8. Аналитика

---

## Зависимости между этапами

```
Этап 0 (подготовка)
    ↓
Этап 1 (MVP: каталог + профиль + заказ) ← основа для всего
    ↓
Этап 2 (посылки)   Этап 3 (возвраты) ← можно параллельно
    ↓
Этап 4 (вишлист + шоурум + уведомления)
    ↓
Этап 5 (рекомендации + ИИ) ← нужен накопленный вишлист
    ↓
Этап 6 (аналитика) ← нужны данные из всех предыдущих этапов
```
