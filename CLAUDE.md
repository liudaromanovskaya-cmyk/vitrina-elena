# CLAUDE.md — Контекст проекта для Claude

## Что это за проект

Проектирование и разработка цифровой системы для малого fashion-бизнеса.
Продавец (Елена) ведёт продажи женской одежды через Telegram.
Цель — добавить структуру поверх живой продажи, не ломая её.

## Кто участвует

- **Елена** — владелец бизнеса, продавец. Работает в Telegram. Не технический пользователь.
- **Людмила** — заказчик системы, анализирует бизнес Елены, ставит задачи.
- **Разработчик** — будет подключён позже.

## Суть бизнеса Елены

- Продаёт женскую одежду (Корея, Турция, Италия) через Telegram-канал
- Товар закупает у поставщиков/складов (таблицы, фото, видео)
- Продажа — через личную переписку, голосовые, подбор вручную
- Есть шоурум в Москве — клиенты приезжают на примерку
- Клиенты — постоянные (годами) и новые (по рекомендации)
- Ассортимент: классика (повторяется годами) + новинки

## Главные проблемы, которые решает система

1. Нет единой карточки заказа — всё в переписке
2. Нет профиля клиента — размеры и предпочтения в голове продавца
3. Нет прозрачных статусов — клиент не знает, где его заказ
4. Нет процесса возврата — только долгие переговоры
5. Нет персональных уведомлений — только общий Telegram-канал
6. Визит в шоурум не фиксируется — данные примерок теряются

## Что система НЕ должна делать

- Заменять живое Telegram-общение Елены
- Превращать бизнес в безликий магазин
- Требовать сложного обучения от клиентов
- Убирать тепло и личность из коммуникации

## Ключевые принципы при работе над проектом

1. **Telegram — канал, не замена.** Живое общение остаётся там.
2. **Голос Елены.** Все автоматические сообщения — тёплые, личные, как от Елены.
3. **Клиент не должен объяснять дважды.** Профиль хранит всё.
4. **Простота для Елены.** Интерфейс должен быть понятен без обучения.
5. **MVP сначала.** Не строить всё сразу — проверять гипотезы поэтапно.

## Основные сущности данных

```
Supplier      — поставщик/склад
Product       — товар (с тегами стиля, повода, сезона)
Client        — клиент (профиль, параметры, предпочтения, события)
Order         — заказ (позиции, статусы, оплата)
Shipment      — посылка (трек, состав, статус)
Return        — возврат (позиции, причина, сумма, статус)
WishlistItem  — отложенный товар (онлайн / примеряла / жду поступления)
FittingSession — сессия примерки в шоуруме
```
